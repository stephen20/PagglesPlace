Background

An authorization passcode is needed to enter the application.
A red box is displayed around the passcode entry area to show the user that they entered an incorrect code.
This is the main view for the application. It opens on the current date and times that the library are closed are "greyed" out.
Once a user clicks on a cell on the grid this form pops up if that space is green (meaning it is open). Form has some fields pre-filled based on the cell clicked.
Confirmation pages allows user to know that the reservation was made.
Reservations that are made on the spot and are currently being occupied are shown with red cells. Reservations that are made for a future time are shown with yellow cells.
When a user clicks on a red cell this pops up showing information about the reservation and giving the user the options to cancel the whole reservation, part of the reservation, or declare that the patron returned his/her keys to the room indicating the reservation was successfully completed.
Similar to clicking a red cell, when a yellow cell is clicked it shows the information and gives the user the option to cancel the whole reservation, cancel part of the reservation, or verify and make the reservation once whoever reserved it shows up to claim their room (which then turns the grid red).
Once a patron returns his/her keys and the user clicks the returned keys button on the application the cells corresponding to this reservation turn blue and the cancel reservation and cancel slot buttons get disabled.
Clicking on the room numbers on the top of the grid brings up this screen that gives information about the room. This allows the user to reserve a room for a patron that meets all of his/her needs.
The user can pick any future date to make reservations for patrons with this easy to use date picker drop down.
The top search bar allows the user to search for patrons and see their past history of rooms that they have reserved and comments that were left. It also utilizes auto fill from the emails that have already been submitted into our database.
This is what the user sees after searching a patron's email. A user can click on the reservation boxes on the left to bring up the information for those specific reservations.
